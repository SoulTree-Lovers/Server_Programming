SQL을 이용하는 프로그래밍의 문제점

1. DBMS 종류에 따라 SQL 문법이 다름
    - ex. MySQL을 사용하다가 Oracle로 변경하면
    - source code도 전부 수정해야 함.

2. Table의 column이 추가, 수정 시 프로그램 코드도 변경해야 함.
    - 객체 class의 속성 추가, 삭제, 수정 등 변경 작업 수행
    - table에 select, insert, update, delete하는 코드 변경

3. 수정할 코드의 양이 많을 경우 프로그램 디버그를 위한 비용 발생
    - 1000개의 코드 수정 -> 개발자 3~4명이 몇 개월 이상 작업
    - 많은 비용 발생!!!

## -> 위 문제들을 ORM으로 해결! ##
